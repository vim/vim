В этом каталоге содержаться командные файлы программы Vim, используемые для
подсветки синтаксиса.

Следующие файлы не предназначены для каких-либо языков, а применяются самим
редактором Vim:

syntax.vim	Применяется при обработке команды ":syntax on". Этот файл
		вызывает командный файл synload.vim.

manual.vim	Применяется при обработке команды ":syntax manual". Этот файл
		вызывает командный файл synload.vim.

synload.vim	Содержит автокоманды на загрузку файла с правилами синтаксиса
		для конкретного языка, когда он был распознан по наименованию
		(расширению) файла. Также в этом файле содержаться комнды для
		выполнения настройки меню «Синтаксис» в графическом интерфейсе.

nosyntax.vim	Применяется при обработке команды ":syntax off". Этот файл
		отменяет загрузку файла synload.vim


Несколько специальных файлов:

2html.vim	Для преобразования любого файла, в котором включена подсветка
		синтаксиса в HTML-файл (только в графическом интерфейсе).
colortest.vim	Для сверки соответствия между наименованием цвета и цветом
		на экране.
hitest.vim	Для просмотра текущих установок подсветки синтаксиса.
whitespace.vim  Для подсветки символов табуляции и пробела.


Если планируете создать файл с описанием правил подсветки синтаксиса для
какого-то языка программирования, то ознакомьтесь с документацией. Наберите для
этого команду ":help usr_44.txt".

Если вы создали файл с описание правил подсветки синтаксиса и это может быть
полезно другим людям, оправьте, пожалуйста, этот файл на электронную почту
Bram@vim.org. Опишите, как выполняется распознавание типов файлов для этого
языка, либо по расширению наименования файла, либо путём проверки нескольких
строк в файле. При написании своего файла с правилами подсветки синтаксиса
руководствуйтесь рекомендациями, описанными в документации, подробнее см.
":help 44.12".

В случае имеющихся замечаний по уже существующему файлу правил подсветки
синтаксиса, эти замечания лучше отправлять сопровождающему данный файл
специалисту. Способ связи с этим специалистом, обычно, указывается в
заголовке самого файла. И только в случае, когда не получен ответ от этого
специалиста, можно отправить свои замечания по адресу bram@vim.org.

В свою очередь, если вы является специалистом, занимающийся поддержкой какого-то
файла с описанием правил синтаксиса и вносите исправления или дополнения в этот
файл, то не забывайте отправлять новые версии файла по адресу bram@vim.org.

Дополнительную информацию см. в документации редактора Vim, ":help syntax".
